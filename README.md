# Домашнее задание 01: Документирование архитектуры в Structurizr

---

## Описание системы

Система предназначена для управления медицинскими записями пациентов, включая регистрацию пользователей, ведение медицинских карт и интеграцию с внешними сервисами.

---

## Функциональные требования

| № | Описание |
|---|----------|
| 1 | Создание нового пользователя |
| 2 | Поиск пользователя по логину |
| 3 | Поиск пользователя по маске имя и фамилия |
| 4 | Регистрация пациента |
| 5 | Поиск пациента по ФИО |
| 6 | Создание медицинской записи |
| 7 | Добавление записи к пациенту |
| 8 | Получение истории записей пациента |
| 9 | Получение записи по коду |

---

## Роли пользователей

| Роль | Описание |
|------|----------|
| Пациент | Просматривает свои записи, регистрируется |
| Врач | Создаёт и просматривает медицинские записи |
| Администратор | Управляет пользователями и справочниками |
| Лаборант | Добавляет результаты анализов |

## Бизнес-кейс (Dynamic Diagram)

**Описание сценария:** Врач создаёт новую медицинскую запись для пациента через веб-интерфейс.

| Шаг | От | Кому | Описание | Протокол |
|-----|-----|------|----------|----------|
| 1 | Врач | Doctor Web Portal | Открывает форму создания записи | HTTPS:443 |
| 2 | Doctor Web Portal | API Gateway | POST /api/records | HTTPS:443 |
| 3 | API Gateway | Medical Records Service | Маршрутизация запроса | HTTPS:443 |
| 4 | Medical Records Service | Medical Records Database | INSERT record | JDBC:5432 |
| 5 | Medical Records Service | Notification Service | Уведомить пациента | HTTPS:443 |
| 6 | Notification Service | Email Service | Отправить email | SMTP:25 |
